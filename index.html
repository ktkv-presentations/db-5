Основные этапы проектирования БД

Определение требований к данным

направлен на сбор информации о том, какие данные будут храниться в базе данных, как они будут использоваться и какие ограничения накладываются на их обработку

Этапы

  • Проведение встреч с пользователями и заинтересованными сторонами
  • Документирование требований в виде спецификаций и диаграмм

Анализ предметной области

включает исследование системы, которая будет использовать БД, для выявления ключевых объектов (сущностей), атрибутов и взаимосвязей между ними

Этапы

  • Выявление всех сущностей и их характеристик
  • Определение связей между сущностями

Концептуальное проектирование

процесс создания абстрактной модели данных, которая описывает основные объекты (сущности) и их взаимосвязи, без учета конкретной СУБД

Этапы

  • Построение ER-диаграммы
  • Определение первичных ключей для каждой сущности

Логическое проектирование

процесс преобразования концептуальной модели в логическую, где сущности превращаются в таблицы с четко определенными атрибутами и связями

Этапы

  • Нормализация данных
  • Определение внешних ключей для создания связей между таблицами

нормализация данных

процесс разбиения данных на таблицы для устранения дублирования и улучшения целостности

Физическое проектирование

процесс преобразования логической модели данных в физическую модель, которая учитывает особенности конкретной СУБД

Этапы

  • Адаптация структуры данных под особенности выбранной СУБД
  • Оптимизация запросов для повышения производительности

Тестирование

проверка корректности работы базы данных перед её внедрением в эксплуатацию


Внедрение

процесс развёртывания базы данных в рабочей среде

  • Создание тестовых данных для моделирования реальной нагрузки
  • Исправление ошибок, выявленных в процессе тестирования

Поддержка и развитие

процесс управления и оптимизации базы данных после её внедрения и внесения изменений в структуру БД по мере необходимости

  • Обеспечение безопасности данных
  • Адаптация базы данных под новые требования и улучшение её производительности

Концепция нормальных форм

Нормальная форма (НФ)

набор правил, которые применяются к таблицам базы данных для улучшения их структуры

Аномалии данных

Аномалия добавления

Аномалия добавления возникает, когда из-за структуры таблицы невозможно добавить новую запись без ненужных данных

Аномалии данных

Аномалия удаления

Аномалия удаления возникает, когда удаление одной записи приводит к потере важной информации

Аномалии данных

Аномалия изменения

Аномалия изменения возникает, когда изменение одной записи требует внесения изменений во множество строк.

Процесс нормализации отношений

Основные этапы нормализации:

  1. Удаление повторяющихся данных
  2. Создание новых таблиц для независимых данных
  3. Установление связей между таблицами с помощью ключей
